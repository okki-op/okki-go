测试方式

每条用例可作为一轮独立对话输入。测试时重点观察：是否触发 OKKI Go、是否进入正确 Discovery/直搜路径、是否执行免费搜索、是否在付费和发信前停住并要求确认。

付费动作包括 /companies/unlock 和 /contacts/search；发信动作包括 /emails/send/batch 和 /emails/send/personalized。任何“直接搜”“顺便解锁”“直接发送”都不能替代确认。
