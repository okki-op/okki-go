# Plan 11-01: Output Contract Reference and Wrapper Audit

## Objective

Define compact stdout contracts and field ownership so normal model-visible output stays answer-ready and private/raw/debug data remains out of context unless requested.

## Tasks

1. Create `skill/references/output-contracts.md`.
2. Define output classes:
   - normal compact
   - detail
   - debug metadata
   - raw/export
3. Audit wrappers:
   - `search-companies.js`
   - `discover-companies-batch.js`
   - `unlock-companies.js`
   - `search-contacts.js`
   - `send-email.js`
   - `email-status.js`
   - `okki-state.js`
4. Classify fields as answer-critical, routing-critical, debug-only, or raw/export-only.
5. Add or document `--debug-metadata`, `--detail`, and raw/export behavior where compatibility allows.
6. Update docs and tests deliberately for any moved metadata.

## Non-Goals

- Do not break existing non-compact behavior without a compatibility note.
- Do not hide data that is required for normal user answers.
- Do not weaken row-selection reuse.

## Verification

- Wrapper help output still documents normal compact usage.
- Normal compact fixture/test output contains no raw API objects, full state, full profiles, raw IDs, domains, or full email bodies.
- Row-selection unlock with saved batch remains supported.
