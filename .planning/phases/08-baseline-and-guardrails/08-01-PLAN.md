# Plan 08-01: Baseline Artifact and Protected Invariants

## Objective

Create the non-regression baseline for the OKKI Go systemic token and latency optimization before any behavior-changing edits.

## Scope

This plan implements Phase 0 from `docs/OKKI_GO_SKILL_SYSTEMIC_TOKEN_LATENCY_OPTIMIZATION_PLAN.md` as roadmap Phase 8. It records the current skill/reference size, representative workflows, protected behaviors, known dirty worktree state, and eval/test assets.

## Tasks

1. Capture current line and byte sizes for:
   - `skill/SKILL.md`
   - `skill/references/*.md`
   - `skill/scripts/README.md`
2. Record representative prompts and expected behavior for:
   - ordinary prospect search
   - row-selection unlock confirmation
   - confirmed unlock
   - result review
   - low-yield diagnosis
   - more/next pagination
   - balance query
   - contact search confirmation
   - email status
3. Mark protected invariants:
   - paid confirmations
   - target-side Chinese-first keyword contract
   - one-primary-field recall-first search
   - compact privacy rules
   - row mapping reuse through latest batch
   - same-language replies
   - warning-only viewed-state write degradation
4. Document existing worktree state relevant to safe execution.
5. Identify current eval/test assets that should protect behavior or need later alignment.
6. Write the baseline to `08-BASELINE.md`.

## Non-Goals

- Do not rewrite `SKILL.md` in this phase.
- Do not change wrapper behavior in this phase.
- Do not run paid OKKI API calls.
- Do not reconcile or revert existing `eval/` deletion/untracked-directory state.
- Do not optimize for a single historical session.
