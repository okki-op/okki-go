# Plan 13-01: Cost-Behavior Evals and Weak-Model Forward-Test Harness

## Objective

Prove the optimization reduces hot-path loading and raw-output exposure without harming search recall, safety, multilingual behavior, or weak-model routing.

## Tasks

1. Add cost-behavior scenarios for the representative prompts from Phase 8.
2. Add static checks for:
   - `SKILL.md` size
   - reference ownership duplication
   - raw-output leakage
   - paid confirmation presence
   - weak-model mode/reference cues
3. Add search-quality comparison against baseline prompts.
4. Add forward-test prompts for fresh or smaller agents that receive only the skill artifact and user-like tasks.
5. Review failures and prefer routing-table, wrapper-field, or command-pattern fixes before adding prose.

## Progress

- Added `skill/scripts/test/prompt-ownership.test.js` to guard against reintroducing prompt obligations for script-owned private/debug fields outside `output-contracts.md`.
- Removed duplicate script-owned field-hiding prose from `SKILL.md`, `search-fast-path.md`, and `paid-actions.md` hot paths, while keeping paid confirmations and output-contract ownership intact.
- Verified the prompt ownership check plus existing compact/debug/source-boundary script tests.

Remaining:

- Add broader cost-behavior scenarios for representative prompts.
- Add or reconcile weak-model mode/reference cue checks.
- Run search-quality comparison and fresh-agent/weak-model forward tests.

## Non-Goals

- Do not encode a fix for one historical session.
- Do not use eval passing to justify weaker paid-action confirmations.
- Do not accept search recall regression as a token optimization.

## Verification

- Search-quality failures block merge.
- Paid-confirmation failures block merge.
- Raw-output leakage failures block merge.
- Weak-model failures produce concrete routing/script fixes or documented follow-up issues.
