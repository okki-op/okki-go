# Plan 10-01: Reference Split, Ownership, and Duplicate-Prose Removal

## Objective

Create or trim references so each normal or advanced mode has one canonical owner and weak models can load only the needed file.

## Tasks

1. Decide whether to create new files (`search-fast-path.md`, `result-review.md`, `search-strategy.md`, `paid-actions.md`) or retain compatible existing filenames with strict ownership.
2. Create `output-contracts.md` placeholder or link target if Phase 11 will fill schemas separately.
3. Add top-level tables of contents to references over 100 lines.
4. Remove duplicated paid-confirmation, keyword-contract, API-parameter, local-state, and field-hiding prose from non-owning references.
5. Replace duplicated details with links to the canonical owner.
6. Keep examples short and mode-specific.

## Non-Goals

- Do not remove advanced capabilities.
- Do not change wrapper stdout yet unless Phase 11 is active.
- Do not turn normal L0 search into Mentor Guided onboarding.
