# Plan 12-01: Script Routing Hints and Command-Pattern Hardening

## Objective

Reduce repeated model reasoning loops by making scripts return small deterministic action hints for common routing decisions.

## Tasks

1. Add or refine `next_action` for wrapper outputs where the next user-visible step is deterministic.
2. Add or refine `health_action` for company discovery where pagination, light recovery, refinement, guided strategy, or expansion routing can be derived from script metadata.
3. Ensure latest-batch row selection uses `--batch latest` without visible `batch_id` in normal output.
4. Add command patterns for L0 search, broad count-based search, next page, confirmed row unlock, result review, balance, contact search, and email status.
5. Preserve recall-first behavior by keeping search-route creativity in the model and pagination/dedupe/budget mechanics in scripts.

## Non-Goals

- Do not increase API filters to save tokens.
- Do not shrink default result counts.
- Do not inspect raw files for normal presentation.

## Verification

- Script tests for `next_action`/`health_action` outputs.
- Manual traces for "continue", "too few", "unlock rows", and "confirm" do not require raw file inspection or re-search when latest batch exists.
